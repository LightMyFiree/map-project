Интерактивная карта природных объектов
1. Что это за проект и почему он классный
Это интерактивная веб-карта природных объектов (горы, водоемы, леса), которая работает полностью в браузере (Chrome, Safari и т.д.).
Главная фишка нашей архитектуры в том, что проект статический.
Что это значит на практике:
•	Нет бэкенда: Нам не нужен сложный и дорогой сервер.
•	Нет базы данных: А значит, она не может "упасть" или быть взломана.
•	Легкий старт: Проект запускается на любом компьютере за 10 секунд без долгих настроек.
•	Бесплатный хостинг: Сайт можно выложить в интернет (например, на GitHub Pages) буквально нажатием одной кнопки.
При этом внешне это полноценное современное приложение: с красивой картой, быстрыми фильтрами, карточками мест, умным построением маршрутов и возможностью загружать свои гео-данные.

2. Из чего состоит проект (простая карта файлов)
Проект делится на две логические части: то, что видит пользователь, и то, что скрыто под капотом.
Часть 1: Фронтенд (лицо сайта)
•	index.html — Скелет. Здесь мы говорим браузеру, где будет лежать карта, где меню, а где кнопки.
•	style.css — Одежда. Здесь прописан весь дизайн (эффекты матового стекла, отступы, поведение на экранах телефонов и переключение темной/светлой темы).
•	script.js — Мозги. Логика загрузки точек, фильтрация, карточки объектов, маршруты и чтение файлов.
Часть 2: Данные (наш внутренний конвейер)
•	data/points.geojson — Рабочий файл. Именно отсюда карта берет все точки прямо сейчас.
•	data_preparation/places.csv — Таблица. Обычная табличка с нашими местами. Ее заполняем вручную.
•	data_preparation/build_map_data.py — Переводчик. Скрипт, который превращает нашу удобную таблицу в формат, понятный карте (GeoJSON).

3. Почему здесь используется Python?
Python нужен здесь не как сервер, а как инструмент для подготовки данных.
Дело в том, что нам удобно заполнять обычные таблички (CSV), а карте нужен строгий программный код (GeoJSON). 
Скрипт работает как умный конвертер:
•	Он позволяет нам комфортно редактировать таблицу.
•	Сам проверяет, не забыли ли мы указать координаты или название (защита от человеческой ошибки).
•	Меняет местами широту и долготу (мы привыкли писать широта-долгота, а стандарт GeoJSON требует долгота-широта).
•	Сохраняет всё в правильной кодировке, чтобы русские буквы не превратились в иероглифы.
Мы разделяем источник данных (где удобно людям) и формат для продакшена (где удобно программе).

4. Быстрый старт (как запустить карту прямо сейчас)
Самый частый сценарий для учебы — запуск на своем компьютере (локально).
1.	Открой терминал (командную строку) в папке с проектом.
2.	Введи команду, которая запустит локальный сервер:
python3 -m http.server 8000
3.	Открой браузер и перейди по адресу: 
http://localhost:8000
Важное правило: Никогда не открывай файл index.html просто двойным кликом (если в адресной строке написано file:///...). Браузер из соображений безопасности заблокирует чтение файла точек, и карта будет пустой. Всегда используй запуск через команду.

5. Как добавить новое место на карту (Шаг за шагом)
В сложный код лезть не придется! Представь, что нам нужно добавить на карту новую гору:
1.	Открой таблицу. Найди файл places.csv в папке data_preparation. Открой его в Excel, Google Таблицах или простом блокноте.
2.	Добавь новую строку. Заполни колонки: id, name, category (water/mountains/forest), lat (широта), lng (долгота), shortDescription (краткое описание), fullDescription (полное описание), image (ссылка на фото) и difficulty (сложность).
o	Важно: Если в твоем описании есть запятые, обязательно возьми весь текст описания в двойные кавычки (например: "Красивая гора, где много снега").
3.	Запусти переводчик. В терминале введи команду:
python3 data_preparation/build_map_data.py
4.	Обнови данные на сайте. Скрипт создал свежий файл points.generated.geojson. Замени им старый рабочий файл:
cp data/points.generated.geojson data/points.geojson
5.	Проверь результат. Обнови страницу в браузере — новая точка уже на карте!

6. Как работают главные фишки
Фишка 1: Мгновенные фильтры
Мы не пересчитываем все точки заново при клике на фильтр. Точки заранее разложены по виртуальным "папкам" слоев. Когда ты нажимаешь кнопку, скрипт просто прячет ненужные папки и оставляет открытой только ту, которую ты выбрала. Карта реагирует моментально.

Фишка 2: Построение маршрутов
Процесс маршрутизации похож на вызов такси:
•	Откуда едем? Браузер запрашивает геолокацию пользователя (радар телефона/ПК). Если пользователь запретил доступ или сидит без GPS, код не падает, а включает план Б — подставляет стартовую точку по умолчанию.
•	Куда едем? Берем координаты из карточки объекта, на который кликнули.
•	Как едем? Передаем эти две точки плагину Leaflet Routing Machine. Он по интернету связывается с сервером дорог, вычисляет повороты и рисует красивую линию поверх карты.
Фишка 3: "Открыть в Яндекс Картах"
Сайт в браузере не умеет проверять, какие приложения установлены на телефоне. Поэтому программисты придумали трюк с таймером:
1.	По клику скрипт пытается открыть специальную ссылку yandexmaps://... (команда телефону открыть приложение).
2.	Запускается таймер на 0.5 сек.
3.	Если приложение есть, оно мгновенно открывается, браузер сворачивается, и таймер замораживается.
4.	Если приложения нет, браузер остается открытым. Через 0.5 секунды таймер срабатывает, код понимает, что "фокус не удался", и открывает обычный сайт Яндекса в новой вкладке.
Фишка 4: Импорт без интернета
Можно загрузить на карту сторонние гео-файлы (ZIP с SHP или KML). Вся распаковка и чтение происходят прямо внутри браузера средствами JavaScript. Файлы никуда не отправляются на сервер, поэтому это работает даже без интернета.

7. Частые ошибки и как их лечить
•	Я изменила таблицу, но на карте ничего не поменялось.
Запускала ли ты скрипт build_map_data.py? Скопировала ли сгенерированный файл в points.geojson? Обновила ли кэш браузера (Ctrl+F5 / Cmd+Shift+R)?
•	Карта грузится, но она пустая, точек нет.
Смотри пункт 4. Ты открыла сайт через file://. Запусти локальный сервер через Python.
•	При запуске Python-скрипта выскакивает ошибка (данные съезжают).
Ты поставила запятую в описании места, но забыла обернуть весь текст в двойные кавычки. В CSV запятая — это разделитель колонок. Исправь текст в таблице.
 
8. Шпаргалка для защиты 
Вопрос: Почему вы использовали Leaflet, а не Яндекс API или Google Maps напрямую?
Ответ: Leaflet — это легкая, бесплатная и открытая библиотека. В отличие от коммерческих API, она не требует платных ключей, не заблокирует карту при превышении лимитов просмотров и дает полный контроль над дизайном и интерфейсом.
Вопрос: Почему в проекте нет базы данных (MySQL) и классического бэкенда?
Ответ: Наша цель — показать современную бессерверную статическую архитектуру. Хранение данных в JSON позволяет загружать сайт мгновенно, не тратить деньги на аренду VPS серверов и реализовать более простое и быстрое решение.
Вопрос: Почему вы не использовали готовые шаблоны дизайна?
Ответ: Цель была сделать интерфейс быстрым и уникальным. Весь дизайн, включая эффекты матового стекла и адаптивность, написан вручную на чистом CSS. Это избавило проект от тяжелого кода, который тянется за крупными библиотеками.
Вопрос: Какие у проекта есть слабые места?
Ответ: Сейчас все объекты загружаются на карту одновременно. Если в базе будет 50 000 точек, слабые смартфоны начнут подтормаживать. Это легко решается внедрением кластеризации маркеров (когда при отдалении камеры близкие точки сливаются в один кружок с цифрой).

9. План развития (если спросят "А что дальше?")
Если проект будут развивать, вот лучшие идеи для доработок:
1.	Добавить ту самую кластеризацию маркеров (Leaflet.markercluster).
2.	Сделать строку текстового поиска по названиям гор и озер.
3.	Добавить в фильтры сортировку маршрутов по уровню сложности.
4.	Настроить CI/CD пайплайн: чтобы при загрузке новой таблицы на GitHub проект сам запускал Python-скрипт и обновлял сайт в интернете.

10. План действий
Весь рабочий цикл добавления данных сводится к 5 шагам:
1.	Заполнить places.csv.
2.	python3 data_preparation/build_map_data.py.
3.	cp data/points.generated.geojson data/points.geojson.
4.	Проверить карту.
5.	Отправить в Git (необязательно): git add ., git commit -m "Добавлены новые точки", git push.
